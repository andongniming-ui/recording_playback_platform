# 内网生产部署清单

本文档用于把录制回放平台从外网开发环境迁移到办公内网。目标是一套代码同时支持外网和内网，通过不同配置、镜像和依赖源切换环境。

## 1. 内网 pip / npm / Maven / Docker 源

建议在内网准备统一制品仓库，例如 Nexus、Artifactory 或公司已有私服。

| 类型 | 模板文件 | 用途 |
|---|---|---|
| pip | `deploy/intranet/pip.conf` | 后端 Python 依赖安装 |
| npm | `deploy/intranet/npmrc` | 前端 npm 依赖安装 |
| Maven | `deploy/intranet/maven-settings.xml` | Java 被测系统和插件构建 |
| Docker | `deploy/intranet/docker-daemon.json` | Docker 镜像源或内网 Registry |

内网正式构建时，使用 `deploy/intranet/.env.production` 里的这些变量切换源：

```bash
PIP_INDEX_URL=http://pypi.intranet.example.com/simple
PIP_TRUSTED_HOST=pypi.intranet.example.com
NPM_REGISTRY=http://npm.intranet.example.com/repository/npm/
PYTHON_BASE_IMAGE=registry.intranet.example.com/library/python:3.11-slim
NODE_BASE_IMAGE=registry.intranet.example.com/library/node:18-bullseye-slim
MYSQL_IMAGE=registry.intranet.example.com/library/mysql:8.0
```

## 2. 固定版本 AREX Agent

当前锁定版本记录在：

```text
deploy/intranet/arex-agent.lock
```

默认版本：

```text
0.4.8
```

内网部署前，把审批过的 jar 放到：

```text
deploy/intranet/third_party/arex-agent/arex-agent-0.4.8.jar
```

然后把实际 SHA256 写回 `arex-agent.lock`，部署前人工核对：

```bash
sha256sum deploy/intranet/third_party/arex-agent/arex-agent-0.4.8.jar
```

平台容器内固定挂载路径是：

```text
/opt/arex/arex-agent.jar
```

被测系统启动时也建议固定使用同一个版本，不要从外网临时下载。

## 3. Docker Compose 内网部署

模板文件：

```text
deploy/intranet/docker-compose.intranet.yml
deploy/intranet/.env.production.example
```

部署步骤：

```bash
cd /home/recording_playback_platform/deploy/intranet
cp .env.production.example .env.production
vi .env.production
./verify-intranet-prereqs.sh
./deploy-intranet.sh
```

如果服务器没有外网，必须提前把下面镜像推到内网 Registry：

```text
python:3.11-slim
node:18-bullseye-slim
mysql:8.0
arex-recorder/backend:1.0.0
arex-recorder/frontend:1.0.0
```

也可以在内网服务器上直接 build，但前提是 pip/npm 的内网源已经准备好。

如果内网没有 pip/npm 私服，也没有 Docker Registry，走完全离线模式：

```bash
# 外网机器
cd /home/recording_playback_platform
deploy/intranet/prepare-offline-bundle.sh /tmp/arex-offline-bundle
deploy/intranet/verify-offline-bundle.sh /tmp/arex-offline-bundle

# 把 /tmp/arex-offline-bundle 和项目代码拷贝到内网机器后
cd /home/recording_playback_platform/deploy/intranet
./import-offline-bundle.sh /path/to/arex-offline-bundle
cp .env.production.offline.example .env.production
vi .env.production
./verify-intranet-prereqs.sh
./deploy-intranet.sh
```

## 4. `.env.production` 内网配置模板

复制模板后重点修改：

```text
PLATFORM_HOST
FRONTEND_PUBLIC_ORIGIN
BACKEND_PUBLIC_ORIGIN
VITE_API_BASE_URL
AR_CORS_ORIGINS
MYSQL_ROOT_PASSWORD
MYSQL_PASSWORD
AR_DB_URL
AR_SECRET_KEY
AR_ADMIN_INIT_PASSWORD
AR_AREX_AGENT_STORAGE_URL
```

注意：

- `AR_SECRET_KEY` 必须是随机 32 位以上字符串。
- `AR_AREX_AGENT_STORAGE_URL` 必须是被测系统机器可以访问的平台后端地址。
- `VITE_API_BASE_URL` 必须是浏览器可以访问的平台后端地址。前端容器启动时会写入运行时 `env.js`，所以离线预构建镜像进入内网后仍可以通过 `.env.production` 修改这个地址。
- 真实 `.env.production` 不要提交到 Git。

## 5. MySQL 初始化和迁移

初始化脚本：

```text
deploy/intranet/mysql/initdb/00-create-platform-database.sh
```

当前平台后端启动时会自动：

1. 创建缺失表。
2. 执行已有增量字段迁移。
3. 校验关键表和字段是否存在。

生产升级建议流程：

```bash
mysqldump -h127.0.0.1 -P3307 -uroot -p --databases arex_recorder > backup_arex_recorder_$(date +%F_%H%M%S).sql
docker compose --env-file .env.production -f docker-compose.intranet.yml pull
docker compose --env-file .env.production -f docker-compose.intranet.yml up -d --build
curl http://127.0.0.1:8000/api/health
```

## 6. 网络端口开通清单

| 方向 | 端口 | 协议 | 用途 | 是否必须 |
|---|---:|---|---|---|
| 用户浏览器 -> 平台前端 | 5173 | HTTP | 访问平台页面 | 必须 |
| 用户浏览器 -> 平台后端 | 8000 | HTTP/WebSocket | API、登录、录制、回放状态 | 必须 |
| 被测系统机器 -> 平台后端 | 8000 | HTTP | AREX Agent 上报录制数据 | 必须 |
| 平台后端 -> 被测系统 | 被测系统服务端口，例如 18080 | HTTP | 连通性测试、回放请求 | 必须 |
| 平台后端 -> 被测系统机器 | 22 | SSH | 平台远程启动/挂载 Agent | 使用 SSH 启动时必须 |
| 平台后端 -> MySQL | 3306/3307 | TCP | 平台数据库访问 | 必须 |
| 运维机器 -> 内网 Docker Registry | 5000/443 | HTTP/HTTPS | 拉取/推送镜像 | 使用内网镜像时必须 |
| 构建机器 -> pip 私服 | 80/443 | HTTP/HTTPS | Python 依赖安装 | 构建时必须 |
| 构建机器 -> npm 私服 | 80/443 | HTTP/HTTPS | 前端依赖安装 | 构建时必须 |
| 构建机器 -> Maven 私服 | 80/443 | HTTP/HTTPS | Java 依赖安装 | 构建被测系统时必须 |

网络验证命令：

```bash
curl http://<platform-host>:8000/api/health
curl http://<tested-system-host>:18080/health
nc -vz <tested-system-host> 22
nc -vz <mysql-host> 3306
```

## 7. 离线依赖包准备

外网机器上可以先执行：

```bash
cd /home/recording_playback_platform
deploy/intranet/prepare-offline-bundle.sh runtime/offline-bundle /path/to/arex-agent-0.4.8.jar
deploy/intranet/verify-offline-bundle.sh runtime/offline-bundle
```

输出目录默认是：

```text
runtime/offline-bundle
```

这个目录可以用于两种方式：

1. 导入内网 PyPI、npm、Maven、Docker Registry，适合长期团队部署。
2. 直接在内网 `docker load` 后用 `.env.production.offline.example` 启动，适合先跑通验证。

长期运行更推荐内网 PyPI、npm、Maven、Docker Registry。
