本文档整理 `/home/recording_playback_platform/systems-under-test/credit-system` 下这套“准入 + 授信”样例系统的接口信息，包含：

## 1. 系统与统一入口

### 主服务

- 系统名：`credit-core`
- 地址：`http://127.0.0.1:29081`
- 统一入口：`http://127.0.0.1:29081/credit/gateway`

### Mock 服务

- 系统名：`credit-mock`
- 地址：`http://127.0.0.1:29083`

### 内部子调用入口

- 风险决策：`http://127.0.0.1:29081/internal/credit/risk`
- 额度定价：`http://127.0.0.1:29081/internal/credit/pricing`

### 统一说明

- 请求方式：`POST`
- Content-Type：`application/xml`
- 请求体：XML
- 返回：XML
- 统一交易码字段：`txn_code`
- 统一链路字段：
  - `tra_id`
  - `request_time`
  - `request_no`

### 日志检索说明

- 主请求、外部 HTTP 子调用、内部 `/internal/credit/*` 调用、数据库查询/更新日志都会带同一个 `traId=...`
- 可直接按 `traId` 检索整笔交易：

```bash
grep 'traId=202604240001004' credit-core/credit-core.log
```

## 2. 主接口总览

## 2.1 CRD_ADMIT 准入

- 接口地址：`http://127.0.0.1:29081/credit/gateway`
- 请求方式：`POST`
- Content-Type：`application/xml`
- 返回：XML
- 交易码：`CRD_ADMIT`

### 请求报文结构

```xml
<request>
  <txn_code>CRD_ADMIT</txn_code>
  <tra_id>...</tra_id>
  <request_time>yyyyMMddHHmmssSSS</request_time>
  <request_no>...</request_no>
  <customer_id>...</customer_id>
  <product_id>...</product_id>
  <id_no>...</id_no>
  <mobile>...</mobile>
  <apply_amount>...</apply_amount>
  <apply_term>...</apply_term>
  <apply_city>...</apply_city>
</request>
```

## 3. 子调用总览

这套系统的子调用分两类：

- 外部 mock 子调用：打到 `credit-mock`
- 内部 HTTP 子调用：打到 `credit-core` 自己的内部接口

## 3.1 外部 mock 子调用

### 1. 征信分查询

- 地址：`GET http://127.0.0.1:29083/mock/credit-score?customerId=...`
- 用途：返回征信分和征信等级
- 典型触发：
  - `CRD_ADMIT`
  - `CRD_LIMIT`

#### 请求示例

```http
GET /mock/credit-score?customerId=C10001
```

#### 返回示例

```json
{
  "customerId": "C10001",
  "creditScore": 78,
  "creditLevel": "B",
  "hitRules": "NO_SERIOUS_OVERDUE"
}
```

### 2. 反欺诈查询

- 地址：`GET http://127.0.0.1:29083/mock/fraud?customerId=...`
- 用途：返回欺诈风险等级和命中标签
- 典型触发：
  - `CRD_ADMIT`
  - `CRD_LIMIT`

#### 请求示例

```http
GET /mock/fraud?customerId=C10005
```

#### 返回示例

```json
{
  "customerId": "C10005",
  "fraudLevel": "MEDIUM",
  "fraudTags": "CONTACT_FREQUENT_CHANGE"
}
```

### 3. 多头借贷查询

- 地址：`GET http://127.0.0.1:29083/mock/multi-loan?customerId=...`
- 用途：返回近 30 天申请次数和借贷平台数
- 典型触发：
  - `CRD_ADMIT` 且 `product_id=P002`
  - `CRD_ADMIT` 且 `apply_amount >= 100000`
  - `CRD_LIMIT` 且 `product_id=P002`
  - `CRD_LIMIT` 且 `apply_amount >= 80000`

#### 请求示例

```http
GET /mock/multi-loan?customerId=C10005
```

#### 返回示例

```json
{
  "customerId": "C10005",
  "loanPlatformCount": 5,
  "applyCount30d": 4
}
```

### 4. 联系人稳定性查询

- 地址：`GET http://127.0.0.1:29083/mock/contact-stability?customerId=...`
- 用途：模拟新客、年轻客户、低收入客户的加强校验
- 典型触发：
  - `CRD_ADMIT`
  - 年龄 `< 25`
  - 或月收入 `< 8000`

#### 请求示例

```http
GET /mock/contact-stability?customerId=C10003
```

#### 返回示例

```json
{
  "customerId": "C10003",
  "stabilityLevel": "MEDIUM",
  "recentChangeCount": 2
}
```

## 3.2 内部 HTTP 子调用

### 1. 内部风险决策

- 地址：`GET http://127.0.0.1:29081/internal/credit/risk`
- 用途：根据征信分、欺诈等级、多头借贷、逾期天数给出风险等级和决策
- 典型触发：
  - `CRD_ADMIT`
  - `CRD_LIMIT`

#### 请求参数

- `traId`
- `txnCode`
- `customerId`
- `productId`
- `creditScore`
- `fraudLevel`
- `multiLoanCount`
- `maxOverdueDays`

#### 请求示例

```http
GET /internal/credit/risk?traId=202604240001004&txnCode=CRD_LIMIT&customerId=C10005&productId=P002&creditScore=66&fraudLevel=MEDIUM&multiLoanCount=5&maxOverdueDays=12
```

#### 返回示例

```json
{
  "txnCode": "CRD_LIMIT",
  "customerId": "C10005",
  "riskLevel": "MEDIUM",
  "decision": "PASS",
  "variantId": "CREDIT_CORE"
}
```

### 2. 内部额度定价

- 地址：`GET http://127.0.0.1:29081/internal/credit/pricing`
- 用途：根据风险等级、月收入、负债率和基础额度计算授信额度
- 典型触发：
  - `CRD_LIMIT`

#### 请求参数

- `traId`
- `txnCode`
- `customerId`
- `riskLevel`
- `monthlyIncome`
- `debtRatio`
- `baseLimit`
- `annualRate`

#### 请求示例

```http
GET /internal/credit/pricing?traId=202604240001004&txnCode=CRD_LIMIT&customerId=C10005&riskLevel=MEDIUM&monthlyIncome=12000.00&debtRatio=2.1667&baseLimit=42000.00&annualRate=12.6000
```

#### 返回示例

```json
{
  "txnCode": "CRD_LIMIT",
  "customerId": "C10005",
  "monthlyIncome": 12000.00,
  "debtRatio": 2.1667,
  "approvedLimit": 10000.00,
  "limitGrade": "C",
  "pricingRate": 13.8000,
  "variantId": "CREDIT_CORE"
}
```

## 6. Postman / ApiPost 文件

- Collection：`credit-system/postman/credit-system.postman_collection.json`
- 本地环境：`credit-system/postman/credit-system.local.postman_environment.json`
