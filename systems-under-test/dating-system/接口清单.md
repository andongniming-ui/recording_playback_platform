# dating-system 接口清单

本文档整理 `/home/recording_playback_platform/systems-under-test/dating-system` 下交友系统被测系统的全部 15 个接口，包含：

- 接口地址
- 请求方式 / 请求格式
- 交易码识别字段
- `trace_id` 链路追踪字段
- 子调用（外呼）
- 数据库调用
- XML 请求示例

## 1. 系统与统一入口

### SAT 环境

- 系统名：`dating-system`
- AREX App ID：`dating-system`
- 地址：`http://127.0.0.1:28090/api/dating/service`

### UAT 环境

- 系统名：`dating-system`
- AREX App ID：`dating-system`
- 地址：`http://127.0.0.1:28091/api/dating/service`

### 统一说明

- 请求方式：`POST`
- Content-Type：`application/xml` / `text/xml` / `text/plain`
- 请求体：XML
- 返回：XML
- 推荐每次请求都带 `<trace_id>`，用于串联主请求、HTTP 子调用、数据库调用日志
- 如果请求里不传 `<trace_id>`，系统会自动生成一个随机 18 位数字，并在响应 XML 中回传
- 交易码字段：`tran_code`

### Postman 导入说明

- Collection 文件：`dating-system/postman/dating-system.postman_collection.json`
- 环境文件：
  - `dating-system/postman/dating-system-sat.postman_environment.json`
  - `dating-system/postman/dating-system-uat.postman_environment.json`
- 导入后选择对应环境即可直接运行
- Collection 已内置 pre-request script，每次发送前会自动生成 18 位 `traceId`
- 请求 XML 中会自动带上 `<trace_id>{{traceId}}</trace_id>`

### 日志检索说明

- 主请求、HTTP 子调用、内部 `/internal/dating/*` 调用、数据库查询/更新日志都会带 `traceId=...`
- 可直接这样查一笔完整交易：

```bash
grep 'traceId=123456789012345678' logs/dating-system.log
```
