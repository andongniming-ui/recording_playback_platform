# Rental System 接口清单

## 基本信息

- BASE 服务：`http://127.0.0.1:20081`
- COMPARE 服务：`http://127.0.0.1:20082`
- 请求方法：全部为 `POST`
- 请求头：`Content-Type: application/xml`，建议同时带 `Accept: application/xml`
- 链路流水号：请求头 `X-Serial-No`，18 位数字；不传时服务端自动生成
- 交易码字段：请求体里的 `trans_code`
- 请求体格式：当前代码实际支持紧凑 `<body>` 格式，不支持旧版 `<request><header>...</header><body>...</body></request>` 包装

## 平台注册建议

| 系统 | 服务端口 | AREX App ID | 交易码提取字段 |
| --- | ---: | --- | --- |
| rental-base | 20081 | `rental-system-base` | `trans_code` |
| rental-compare | 20082 | `rental-system-compare` | `trans_code` |

启动脚本默认使用：

```bash
AREX_AGENT_JAR=/home/test/arex-platform_v1/backend/arex-agent/arex-agent.jar
AREX_STORAGE_HOST=127.0.0.1:8000
AREX_RECORD_RATE=100
```

如果平台不在本机，把 `AREX_STORAGE_HOST` 改成平台机器的局域网 IP 和端口，例如 `192.168.43.64:8000`。

## 调用示例

```bash
curl -sS -X POST 'http://127.0.0.1:20081/api/vehicle/query' \
  -H 'Content-Type: application/xml' \
  -H 'Accept: application/xml' \
  -H 'X-Serial-No: 202604290000000001' \
  -d '<body><trans_code>VEH002</trans_code><vehicle_id>1</vehicle_id></body>'
```

响应示例：

```xml
<response>
  <header>
    <serial_no>202604290000000001</serial_no>
    <ret_code>0000</ret_code>
    <ret_msg>查询成功</ret_msg>
  </header>
  <body>...</body>
</response>
```

## 内部子调用

业务接口内部会调用这些地址：

- `GET /api/internal/insurance/query`
- `GET /api/internal/store/info`
- `GET /api/internal/risk/check`
- `GET /api/internal/pricing/calculate`
- `POST /api/internal/notification/send`
- `GET /api/internal/payment/gateway`
- `POST /api/internal/payment/refund`
- `GET /api/internal/reconciliation/query`
- `GET /api/internal/report/enrich`

这些地址是被测系统内部子调用，不应该作为平台测试用例库里的主入口用例。平台后端已把 `/api/internal/*` 识别为内部 Servlet 记录，实时录制数会排除它们。

## 已知验证点

- `rental-base` 与 `rental-compare` 存在刻意差异，用于回放对比验证。
- 项目使用 `JdbcTemplate`，AREX 0.4.8 对数据库子调用录制能力有限；主 Servlet 调用和 HTTP 子调用可用于当前平台录制回放验证。
- 推荐使用 `send_samples.sh` 或 Postman 集合作为准入样例，二者已和真实代码字段对齐。
