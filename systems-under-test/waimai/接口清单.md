# waimai 接口清单

本文档整理 `/home/recording_playback_platform/systems-under-test/waimai` 下两个被测系统的全部 30 个逻辑接口，包含：

- 接口地址
- 请求方式 / 请求格式
- 交易码字段
- `traId` 链路追踪字段
- 子调用
- 数据库调用
- JSON / XML 请求示例

## 1. 系统与统一入口

两个系统对外入口完全一致，只是端口和配置不同：

### 系统 base

- 系统名：`waimai-base`
- 地址：`http://127.0.0.1:19091`
- 主网关 JSON：`http://127.0.0.1:19091/waimai/gateway/json`
- 主网关 XML：`http://127.0.0.1:19091/waimai/gateway/execute`

### 系统 compare

- 系统名：`waimai-compare`
- 地址：`http://127.0.0.1:19092`
- 主网关 JSON：`http://127.0.0.1:19092/waimai/gateway/json`
- 主网关 XML：`http://127.0.0.1:19092/waimai/gateway/execute`

### 统一说明

- JSON 网关请求方式：`POST`
- JSON 网关 Content-Type：`application/json`
- JSON 网关返回：JSON
- XML 网关请求方式：`POST`
- XML 网关 Content-Type：`application/xml`
- XML 网关返回：XML
- 交易码字段：`txnCode`
- 参数字段：`params`
- `traId` 不支持像 `didi` 一样从请求里透传，当前由服务端在每次请求开始时自动生成
- `requestTime` 也是服务端自动生成
- `traId` 和 `requestTime` 会出现在网关响应里，内部子调用和数据库日志也会使用同一个 `traId`

### Postman 导入说明

- Collection 文件：`waimai/postman/waimai.postman_collection.json`
- 环境文件：
  - `waimai/postman/waimai-base.local.postman_environment.json`
  - `waimai/postman/waimai-compare.local.postman_environment.json`
- 导入后选择对应环境即可直接运行

### 日志检索说明

- 主请求、内部 `/waimai/internal/*` 调用、数据库查询/更新日志都会带同一个 `traId`
- 可直接按日志里打印出的 `traId` 检索整笔交易：

```bash
grep '\[f3f9b6a6c0fe4d7db5a1\]' waimai/waimai-base/waimai-base.log
```

## 2. 复杂接口统一调用链

以下接口属于“复杂接口”，会产生子调用和数据库访问：

- `PLACE_ORDER`
- `CONFIRM_ORDER`
- `CANCEL_ORDER`
- `QUERY_ORDER`
- `APPLY_REFUND`
- `SEARCH_MERCHANT`
- `MERCHANT_DETAIL`
- `ADD_CART`
- `QUERY_CART`
- `SUBMIT_REVIEW`
- `RIDER_LOCATION`
- `RECHARGE_WALLET`
- `WITHDRAW_WALLET`
- `QUERY_WALLET`
- `MERCHANT_SETTLE`

这些复杂接口会按交易码触发不同的 DB 操作和内部 HTTP 子调用。

### HTTP 子调用

- `POST /waimai/internal/pricing`
- `POST /waimai/internal/discount`
- `POST /waimai/internal/risk`
- `POST /waimai/internal/delivery-time`

说明：

- 子调用是系统内部通过 `RestTemplate` 发起
- 目标仍是本服务本机端口
- 录制时如果后面给 `waimai` 挂上 AREX Agent，这些请求可以作为 HTTP 子调用被采集

### 数据库调用

读操作：

- `SELECT * FROM orders WHERE order_id = ?`
- `SELECT * FROM merchants WHERE name LIKE ?`
- `SELECT * FROM merchants WHERE merchant_id = ?`
- `SELECT * FROM cart WHERE customer_id = ?`
- `SELECT * FROM riders WHERE rider_id = ?`
- `SELECT * FROM wallets WHERE customer_id = ?`

写操作：

- `INSERT INTO orders(...) VALUES (...)`
- `UPDATE orders SET status = ? WHERE order_id = ?`
- `UPDATE products SET stock = stock - 1 WHERE product_id = ?`
- `UPDATE products SET stock = stock + 1 WHERE product_id = ?`
- `INSERT INTO refunds(...) VALUES (...)`
- `INSERT INTO cart(...) VALUES (...)`
- `INSERT INTO reviews(...) VALUES (...)`
- `UPDATE wallets SET balance = balance + ? WHERE customer_id = ?`
- `INSERT INTO settlements(...) VALUES (...)`

### 相关业务表

- `customers`
- `merchants`
- `products`
- `orders`
- `order_items`
- `riders`
- `deliveries`
- `reviews`
- `wallets`
- `settlements`
- `refunds`
- `cart`

## 4. 接口明细

## 4.1 PLACE_ORDER 下单

- 接口地址：
  - base：`http://127.0.0.1:19091/waimai/gateway/json`
  - compare：`http://127.0.0.1:19092/waimai/gateway/json`
- 请求方式：`POST`
- 交易码字段：`txnCode`
- 子调用：
  - `POST /waimai/internal/pricing`
  - `POST /waimai/internal/discount`
  - `POST /waimai/internal/risk`
- 数据库调用：
  - 写入 `orders`
- 备注：最适合做录制回放样本，既有 DB 写入，也有 3 个 HTTP 子调用
- JSON 请求示例：

```json
{
  "txnCode": "PLACE_ORDER",
  "params": {
    "customerId": "C001",
    "merchantId": "M001",
    "orderId": "ORD_00001",
    "productId": "P001",
    "basePrice": "30.0",
    "originalAmount": "50.0",
    "amount": "100.0",
    "riderId": "R001"
  }
}
```

## 6. 推荐优先录制的交易码

如果你想在 `arex-recorder` 里看到更完整的录制链路，建议优先录以下交易码：

- `PLACE_ORDER`
- `QUERY_ORDER`
- `SEARCH_MERCHANT`
- `QUERY_CART`
- `RIDER_LOCATION`
- `RECHARGE_WALLET`

原因：

- 既有 JDBC 子调用
- 也有 HTTP 子调用
- base / compare 默认存在受控差异
- 比简单交易更适合观察录制、回放、子调用比对、响应差异
